# wiki/resources/ — curated, trusted recommendations

The coach's **recommendation engine**. When a PM asks "what should I read/watch about X?", the coach answers from **trusted sources** — things [[marty-cagan|Marty Cagan]] and the [[svpg|SVPG]] team explicitly recommend — matched to the **problem or topic** at hand.

What lives here:

- **[[recommended-library]]** — the hub: a problem/topic → resource map (books, videos, SVPG articles), each with its **provenance** (who recommended it, with a link) so the coach only ever recommends from trusted sources.

Conventions:

- **Books** are full nodes in [`../entities/books/`](../entities/books/); the hub links to them. Each book page records *why SVPG/Cagan recommends it*, *what topic/problem it helps with*, and a link.
- **Videos and external articles** are listed in [[recommended-library]] with direct links and topic tags (no separate page needed unless one earns it).
- **Always carry provenance.** Every recommendation must trace to a trusted source (e.g. *TRANSFORMED*'s further-reading, the SVPG "Product Operating Model: An Introduction" resource list, or a Cagan/SVPG article). Never recommend something as "Cagan-endorsed" without that trail.
- Tag each item by the [[overview|model area]] / topic it serves (teams, strategy, discovery, delivery, culture, leadership/coaching, company exemplars, foundations) so the coach can match a problem to a resource.
