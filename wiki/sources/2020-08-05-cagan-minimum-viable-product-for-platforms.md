## Notes for the coach
- **Platform products vs developer tools — a real distinction.** In developer *tools*, developers = users = customers. In platform *services/APIs*, developers are **not** the users/customers — they're partners; the real users are the end users (e.g., the buyer using a payments API).
- **The single biggest mistake platform teams make:** optimizing for the developer at the expense of the end-user experience. (Flash: poor DX, great end-user XP → it grew.) Optimize the end-user experience *over* developer experience — without making DX bad.
- **The four kinds of prototypes:** (1) low-fidelity simulation (wireframe), (2) high-fidelity simulation (looks real, fake data), (3) **live-data prototype** (real data ± real traffic; a fraction of the work — no automated tests / SEO / performance / scale), (4) **feasibility prototype** (technical/performance risk).
- Reinforces: two inconvenient truths; two-week rule / Facebook "don't fall in love rule"; dual-track agile; data beats opinions; MVP = a test not a product; reference customers = the early-stage MVP proof; instrument everything ("if you're not instrumenting, you don't deserve any investment"); Alan Cooper's *The Inmates Are Running the Asylum*.
