# wiki/principles/ — the first principles of the model

One page per first principle. SVPG frames the Product Operating Model as a set of first principles (organized around how you decide which problems to solve, how you solve them, and how you build) rather than a fixed process. Capture each as its own page so the coach can hold you to the principle, not the ritual.

Use the [`principle`](../../templates/principle.md) template.

A good principle page: the principle stated plainly, the underlying belief / why it holds, what violating it looks like in practice, and links to the [`concepts`](../concepts/), [`competencies`](../competencies/), and [`diagnostics`](../diagnostics/) it touches.
